# Developer Inventory Plan

Before writing full reference pages, maintain an inventory of all known developer-facing symbols.

## Inventory goals

- Identify every known object, method, property, event, enum, and SDK entry point.
- Classify each by surface.
- Attach evidence level.
- Mark documentation status.
- Track what still requires research.

## Inventory folders

```text
inventory/
├── extendscript/
│   ├── objects.csv
│   ├── methods.csv
│   ├── properties.csv
│   └── enums.csv
├── uxp/
├── cep/
├── qe/
└── sdk/
```

## Inventory before prose

Do not write a long chapter before the corresponding inventory exists. The inventory prevents omissions and duplicates.
