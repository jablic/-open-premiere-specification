# Media Import & Ingestion

## TL;DR

**Import = adding media files to Premiere project.** Two modes: **Linked** (references original file) vs **Proxy** (low-res substitute). **Automation:** Limited via ExtendScript; better via UXP (25.6+). **Gotcha:** Import doesn't copy files — only creates references.

---

## Linked vs Proxy

### Linked Media

- References original file (no copy)
- High quality, full resolution
- If file moves, reference breaks
- Best for: Final edit, color grading

### Proxy Media

- Lower resolution copy (generated by Premiere)
- Faster playback, less CPU
- Original stays in place
- Switch to original for export
- Best for: Rough cut, multicam, 4K+

```javascript
// ExtendScript: Import with auto-proxy
var file = new File("/path/to/video.mov");
var binItem = app.project.importFile(file);
binItem.createMediaProxy("1/4 Resolution");
```

---

## Import API (Limited)

### ExtendScript

```javascript
var file = new File("/Volumes/media/clip.mov");
if (file.exists) {
  app.project.importFile(file);
}
```

**Limitations:** No batch import, no proxy automation, no link validation.

### UXP (Emerging)

```javascript
const { application } = require("premierepro");

(async () => {
  const proj = await application.activeProject;
  await application.executeTransaction(async () => {
    const item = await proj.importFile("/path/to/clip.mov");
    const name = await item.name;
    console.log("Imported:", name);
  });
})();
```

## Link Management

### Relink Offline Media

When media moved:
1. Window → Media Browser → Offline Files
2. Right-click → Relink
3. Navigate to new location

**No API:** Cannot automate relinking via script (reverse-engineering research ongoing).
