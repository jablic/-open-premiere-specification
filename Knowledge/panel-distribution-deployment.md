# Panel Distribution & Deployment

## TL;DR

**CEP panels:** Self-signed cert (dev), Adobe-signed cert (distribution). **UXP plugins:** Built via Adobe's plugin marketplace or self-hosted. **Code signing:** Required on macOS 25.2.3+ (Gatekeeper), optional on Windows. **Versioning:** Semantic versioning (major.minor.patch). **Distribution:** Marketplace (official), GitHub releases (community), direct links (internal).

---

## Code Signing (CEP Panels)

### Self-Signed Certificate Generation

**macOS:**
```bash
# Generate self-signed certificate valid for 365 days
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes

# Fill in prompts:
# Country: US
# State: CA
# Locality: San Francisco
# Organization: Your Company
# Org Unit: R&D
# Common Name: Your Name
# Email: you@company.com

# Create PKCS12 bundle (for Windows signing tool)
openssl pkcs12 -export -out cert.p12 -inkey key.pem -in cert.pem
```

**Windows:**
```powershell
# Using Windows built-in certutil or third-party tool
# MakeCert (deprecated but still available):
MakeCert -sv MyKey.pvk -cy end -n "CN=Your Name" -a sha256 -len 2048 -r MyKey.cer

# Convert to PFX
pvk2pfx -pvk MyKey.pvk -spc MyKey.cer -pfx MyKey.pfx
```

### Sign CEP Panel (macOS)

```bash
# Use Adobe's ZXPSignCmd tool
./ZXPSignCmd -sign panel_folder/ output.zxp cert.p12 password timestamp_server

# Example with Adobe's timestamp server
./ZXPSignCmd -sign MyPanel/ MyPanel.zxp mycert.p12 mypassword \
  http://timestamp.adobe.com/
```

### Sign CEP Panel (Windows)

```batch
# Using ZXPSignCmd.exe
ZXPSignCmd.exe -sign panel_folder\ output.zxp cert.p12 password timestamp_server

REM Example
ZXPSignCmd.exe -sign "C:\MyPanel" "C:\MyPanel.zxp" mycert.p12 mypassword ^
  http://timestamp.adobe.com/
```

**Download ZXPSignCmd:**
- https://github.com/Adobe-CEP/CEP-Resources/releases

## UXP Plugin Packaging

### UXP Project Structure for Distribution

```
my-uxp-panel/
├── src/
│   ├── index.js
│   ├── index.html
│   └── style.css
├── package.json
├── manifest.json
```


## Versioning Strategy

### Semantic Versioning

**Version Format:** `MAJOR.MINOR.PATCH`

- **MAJOR** (1.0.0 → 2.0.0): Breaking changes, incompatible API
- **MINOR** (1.0.0 → 1.1.0): New features, backward compatible
- **PATCH** (1.0.0 → 1.0.1): Bug fixes

**Examples:**
- `1.0.0` — Initial release
- `1.1.0` — Added caption import feature
- `1.1.1` — Fixed crash on empty sequences
- `2.0.0` — Dropped ExtendScript, UXP-only (breaking)

### Update Manifest

```javascript
// manifest.json or CHANGELOG.md
{
  "version": "1.2.0",
  "changelog": [
    {
      "version": "1.2.0",
      "date": "2026-07-01",
      "changes": [
        "Added batch marker deletion",
        "Fixed marker color not persisting",
        "Improved performance for 1000+ clip sequences"
      ],
      "notes": "Requires Premiere 25.6+"
    },
    {
      "version": "1.1.0",
      "date": "2026-06-15",
      "changes": [
        "Added caption SRT export",
        "New: Regex clip filtering"
      ]
    }
  ]
}
```

## Distribution Channels

### Channel 1: Adobe Extension Marketplace (Official)

**Requirements:**
- Adobe CC account
- Signed application (code signing certificate)
- Privacy policy URL
- Support email
- Minimum test score (quality assurance)

**Process:**
1. Create Adobe account + developer profile
2. Upload plugin to https://exchange.adobe.com/
3. Pass automated tests + manual review
4. Get listed in official marketplace
5. Automatic updates via Adobe's system

**Advantages:**
- Official channel (users trust it)
- Automatic update mechanism
- Adobe handles distribution/hosting
- Discovery via marketplace

**Timeline:** 2-4 weeks for review

### Channel 2: GitHub Releases (Community)

**Setup:**
```bash
# Create GitHub release with asset
git tag -a v1.0.0 -m "Release version 1.0.0"
git push origin v1.0.0

# Upload ZXP or UXP plugin to release
# Users download from: https://github.com/yourname/panel/releases
```

**Distribution:**
```bash
#!/bin/bash
# Automated release script

VERSION="1.0.0"
FILE="MyPanel.zxp"

# Create release on GitHub
gh release create v${VERSION} \
  --title "Release v${VERSION}" \
  --notes "Bug fixes and new features" \
  ${FILE}

echo "Published to GitHub Releases"
```

**Advantages:**
- Free hosting
- Version control integration
- Easy to maintain
- Direct user download links

### Channel 3: Direct Links (Internal/Enterprise)

**Host on Company Server:**
```html
<!-- Installation instructions HTML -->
<h2>Panel Installation</h2>
<ol>
  <li>Download: <a href="https://company.com/panels/MyPanel.zxp">MyPanel.zxp</a></li>
  <li>Place in: ~/Library/Application Support/Adobe/CEP/extensions/ (Mac)
              or %APPDATA%\Adobe\CEP\extensions\ (Windows)</li>
  <li>Restart Premiere</li>
  <li>Window → Extensions → My Panel</li>
</ol>

<h3>Current Version: 1.2.0</h3>
<p>Released: 2026-07-01</p>
```

**Advantages:**
- Full control
- No app store review delays
- Works offline
- Enterprise distribution

## Installation Paths

### CEP Panel Manual Installation

**macOS:**
```bash
# Copy signed ZXP to extensions folder
cp MyPanel.zxp ~/Library/Application\ Support/Adobe/CEP/extensions/

# Or extract directly
unzip MyPanel.zxp -d ~/Library/Application\ Support/Adobe/CEP/extensions/MyPanel
```

**Windows:**
```cmd
REM Copy to extensions folder
xcopy MyPanel.zxp "%APPDATA%\Adobe\CEP\extensions\" /Y

REM Or extract with 7-Zip (ZXP is ZIP format)
7z x MyPanel.zxp -o"%APPDATA%\Adobe\CEP\extensions\MyPanel"
```

### UXP Installation (Marketplace)

Users install via:
1. Premiere Pro → Window → Extensibility
2. Search for your plugin
3. Click Install
4. Automatic updates handled

### Version Update Strategy

```javascript
// Check for updates in panel
function checkForUpdates() {
  var currentVersion = "1.1.0";
  var latestURL = "https://company.com/panels/latest-version.json";
  
  fetch(latestURL)
    .then(r => r.json())
    .then(data => {
      if (data.version > currentVersion) {
        showNotification("Update available: " + data.version);
        // Prompt user to visit download page
      }
    })
    .catch(e => console.error("Update check failed", e));
}

checkForUpdates();
```

- ☐ Release notes written
- ☐ Marketplace metadata prepared (icon, screenshots, description)
- ☐ Support email/contact configured
- ☐ Privacy policy link available
- ☐ Build artifacts generated and tested
- ☐ Git tag created
- ☐ Released to chosen distribution channel
